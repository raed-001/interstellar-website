## Folder Structure & Filing Rules

All files saved to this workspace must follow the structure below. Do not save files loose at the root level — always file them in the correct subfolder.

### 2.0 Contracts/
All contracts, agreements, NDAs, MOUs, and related legal documents, organised by counterparty.

**One subfolder per counterparty.** Named clearly by company name.

```
2.0 Contracts/
├── Nibras Technologies/
│   ├── (signed agreements go here — no version suffix once signed)
│   └── Drafts/       ← working drafts and superseded versions only
├── North Star/       ← Northstar Telecom Technology WLL + HFCL/iO business
├── Awaser/
├── VOda FOne/        ← Vodafone Oman
├── OMNICOMM/
├── PayMob/
├── Speridian/
├── Spider Silk/
├── Cloud Acrop/
├── Data Mount/
├── Fyient - E Invoicing/
├── KNITT/
└── Sama X/
```

**Filing rules for contracts:**
- Signed/executed documents → root of the counterparty folder
- Working drafts and versions → `Drafts/` subfolder within that counterparty folder
- Naming convention: `[Counterparty]_[DocumentType]_[Status]_[Date].docx/.pdf`
  - Example: `Nibras_Agreement_VersionA_Agency_Model.docx`
  - Example: `Agency Agreement NS | IS Signed Jan 2025.pdf`
- Always keep the signed PDF alongside the working DOCX
